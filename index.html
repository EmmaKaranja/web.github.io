TERMINOLOGIES

  • Web architecture:

    • The art and science of structuring and classifying websites and intranets to help users find and manage information.

  • Internet:

    • An infrastructure that allows billions of computers to be connected and services such as the web and email are built on it.

  • Web:

    • This is a service built on top of the infrastructure/ internet.

  • Web Server:

    • A computer that hosts a website on the internet.

  • Web browser:

    • A software e.g., Mozilla Firefox, Google Chrome, Opera, Microsoft Internet Explorer or Safari that that retrieves and displays webpages.

  • Web Page:

    • A document that is displayed on the web browser.

  • Web Site:

    • A collection of web pages that are grouped together and usually connected in different ways.

  • Search Engine:

    • A web service/ special kind of website e.g. Google or Bing that helps find web pages (MDN Web Docs. 2005).

HISTORY OF THE INTERNET AND THE WEB

The first internet activity dates to 29th October 1969 through the Interface Message Processor (IMPs). It was referred to as ARPAnet named after the US Defense Department's Advanced Research Projects Agency. They developed the Transmission Control Protocol and the Internet Protocol (TCP/IP) to run the following applications:

  • Telnet to connect a local terminal to remote computers.
  • File Transfer (FTP) for moving files between computers.
  • Simple Mail Transfer Protocol (SMTP) for E-mail/ exchanging messages.

Other notable milestones:

  • Ethernet, an example of Local Area Network (LAN) was developed in 1973.
  • Mosaic, a user-friendly web browser made it possible for millions of users to access the World Wide Web in the mid-1990s (Kefer and Baiget, 2001).

WEB ARCHITECTURE MAIN COMPONENTS

  1. Presentation layer:

    2. It is accessible to user through the browser and consists of the User Interface (UI) components that support interaction with the system. It is developed using:

    HTML: The code that determines what the website will contain.
    CSS: Determines how the website will look.
    JavaScript: Make the website interactive by responding to a user’s actions.

  2. Business layer:

    3. It is also known as the Business Logic/ Domain Logic/ Application Layer. It accepts user requests from the browser, processes them, and determines the routes through which the data will be accessed.

  3. Persistence layer:

    4. It is also known as storage/ data access layer. It is closely connected to the business layer, so the logic knows which database to talk to and the data retrieving process is more enhanced (Alexsoft n.d.).

IMPORTANCE OF WEB ARCHITECTURE

  1. Helps to understand the needs of the users and provide them with answers.
  2. Saves time for the users.
  3. Makes updating and maintenance work simplified.
  4. Makes websites look more aesthetic.
  5. Reduces overall cost since the maintenance cost will also be less (Rosenfeld and Morville, 2002).

WEB PAGE DELIVERY

For a web page content information to be presented to the user, the following are required for the system:

  1. Retriever engine:

    2. Transmits a domain to a server(parser) and the server sends the information needed back to the retriever.

  2. Parser engine:

    3. Analyses the HTML source code of a web page and determines whether to display information.

  3. Display engine:

    4. Displays the approved information from the Parser (Pennell et al., 2006).

Step by step explanation:

  • First the device must be connected to a router to make sure the message arrives at the right destination.
  • Next, a modem is needed to connect to a network.
  • Next the network is connected to an Internet Service Provider (ISP) so that the message is carried from the original network to the destination network.
  • Type the web address/ Uniform Resource Locator (URL) in the address bar of the browser.

    • By default, most of the URLs start with the protocol identifier/scheme; HTTP and HTTPS for forms/logins to prevent information from being intercepted but other protocols include: mailto (for mail client) and ftp (file transfer).
    www.gooogle.com is an example of a domain name/host and has a unique IP address assigned to it but not user friendly so the Domain Name System (DNS) finds the proper IP address of the intended domain name.
    Next bit is the path to the file e.g., /US/docs/Learn.
    Another component is, The Query String; Sometimes the following example can be displayed: ‘?key1=value1&key2=value2’. These are extra parameters provided to the Web server and are a list of key/value pairs separated with the & symbol. Thus, displaying more options.
    A fragment identifier/ anchor may also be present in the URL and starts with the symbol ‘#’ to indicate somewhere in the document/ subsection. It is never sent to the server with the request.
    The scheme and host components are not case-sensitive, but the path and query string are case-sensitive. Typically, the whole URL is specified in lowercase.

    The scheme is followed by a colon and two forward slashes.
    The path name begins with a single forward slash.
    If a query string is specified, it is preceded by a question mark (IBM 2014).
